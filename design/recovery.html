🔧 安全恢复与救砖说明

Failsafe、U-Boot 与升级注意
🚨 Failsafe 模式

开机过程中在特定时机触发 failsafe，可进入最小系统以重置 root 密码或恢复网络配置。不同机型按键与时机不同，请以 OpenWrt Failsafe 文档 与当前设备在 OpenWrt 上的 Wiki 页为准。

  • 通常需在上电后 LED 闪烁阶段反复按 Reset 或指定键。
  • 进入后可通过以太网固定 IP 访问（通常为 192.168.1.1）。
  • 执行以下命令可清除覆盖层配置（会丢失已改配置）：
firstboot -y && reboot
💾 U-Boot / TFTP 恢复

RAX3000M / ARX3000M 请以本机型的 OpenWrt 设备页或厂商救砖教程为准；此处仅列通用思路：

  • 上电中断进入 U-Boot（常通过串口或按住复位键上电）。
  • 使用 TFTP / Web 恢复写入 factory 或 initramfs 镜像，具体命令因 Bootloader 而异。
⚠️ 具体命令因 Bootloader 而异，请以本机型文档为准。错误操作可能导致设备变砖，请谨慎操作。
⬆️ sysupgrade 与保留配置

跨大版本或更换分支升级时，保留配置可能导致 UCI 与新版默认值不兼容。若升级后异常，可在 failsafe 或首次启动失败时尝试不保留配置重刷，或手动清理 /etc/config 冲突项。

建议：跨大版本升级时使用 -n 参数不保留配置：

sysupgrade -n /tmp/firmware.bin
⚠️ 使用 -n 参数将清除所有自定义配置，升级前请备份重要设置。
