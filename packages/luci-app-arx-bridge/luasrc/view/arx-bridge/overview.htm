上行 / 桥接

本页汇总当前 WAN、无线客户端（STA）、relayd、mwan3、travelmate 的只读状态，并提供标准 LuCI 入口。 修改接口请在「接口 / 无线」中操作；本插件不直接写入 network / wireless。

双频分工（建议）

若需连接上级 WiFi 同时为本机提供热点，优先让一路射频做 STA 连上级、另一路开 AP， 以减少同频 AP+STA 的信道绑定与兼容性问题。是否支持以 mt76 与固件版本为准，需在设备上实测。

NAT 路由 vs relayd（伪桥）

  • NAT：无线或接口作为 WAN，LAN 为私网，兼容性最好，适合大多数上级热点。
  • relayd：LuCI 中「中继桥」类场景，便于与上级同网段类需求；需正确配置防火墙区域与 DHCP/IPv6。

Captive Portal（网页认证）

酒店、校园等需浏览器登录的网络，透明二层桥往往难以完成认证。使用 NAT 路由时，可先保证路由器自身能上网， 再由下级设备浏览器完成认证；若仅路由器需弹窗，请用连接本机 LAN 的终端访问管理地址或按上级提示操作。

多上行与备份

有线 WAN 与 WiFi 上行可同时存在，通过 mwan3 配置优先级、metric 与离线检测，实现故障切换（需自行完成 mwan3 规则）。
