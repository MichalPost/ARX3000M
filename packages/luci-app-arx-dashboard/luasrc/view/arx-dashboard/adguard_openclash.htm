本页能做什么

参考社区教程整理的链路：客户端 → dnsmasq（53）→ AdGuard Home（自定义端口）→ OpenClash DNS（常见 7874）。 OpenWrt 侧可自动完成的只有「在 dnsmasq 增加/删除 127.0.0.1#端口 转发」；AdGuard 与 OpenClash 的界面选项因版本而异，须在各自 LuCI 中按下列说明勾选。

思路来源：Actions-CMCC-RAX3000M · adguard-openclash.md

添加 dnsmasq 转发

请先在 AdGuard Home 中将「重定向」设为作为 dnsmasq 的上游，并把监听端口设为下方填写的端口（勿用 53）。

移除 dnsmasq 转发

填入要移除的端口号，等价于在网络 → DHCP/DNS → DNS 转发中删除对应条目。提交后会执行 dnsmasq restart。

AdGuard Home 手动步骤

  1. 管理设置 → 重定向：作为 dnsmasq 的上游；监听端口使用非 53（与上方表单一致）。
  2. 设置 → DNS 设置 → 上游 DNS 服务器：填入 127.0.0.1:7874（OpenClash 默认 DNS 端口；若改过 OpenClash DNS 端口，请填实际端口）。
  3. 若上网异常，可尝试将 Bootstrap DNS 设为运营商 DNS。

OpenClash 手动步骤

  1. 运行模式优先 Redir-Host；不行再试 Fake-IP。
  2. 插件设置 → DNS：关闭本地 DNS 劫持。
  3. 覆写设置 → DNS：勾选自定义上游 DNS；取消两个「追加 … DNS」类选项（若曾勾选）。
  4. 有私有 DNS 可在覆写中填写；否则可保持默认。

部分设备不通（如 iOS）

可在该 WiFi 上把 DNS 手动设为路由器 LAN IP。其它情况可用 Android 查看当前 DNS 是否指向路由器。
