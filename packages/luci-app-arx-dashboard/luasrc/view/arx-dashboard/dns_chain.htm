本地 DNS 解析链

谁在回答 53 端口

典型路径（示意）

  1. 内网客户端使用 DHCP 下发的 DNS（多为路由器 LAN 地址）。
  2. dnsmasq：OpenWrt 默认 DNS/DHCP，可转发到上游或按域名分流。
  3. SmartDNS：若监听 53 或 dnsmasq 将上游指向其端口，则查询经 SmartDNS。
  4. AdGuard Home：若独占 53 或作为 dnsmasq 上游，则广告过滤在此层完成。
  5. 最终向 WAN 侧解析服务器或 DoH/DoT 出口查询。

实际顺序取决于 UCI（dhcp、smartdns、端口转发、iptables REDIRECT）以及各服务是否启用。
